# SocialHub

> Социальная платформа для осознанного потребления контента с управляемой лентой и прозрачной модерацией.

---

## О проекте

**SocialHub** — учебный проект по предмету *Архитектура программных систем*.

Цель: создать масштабируемую социальную платформу, в которой пользователь получает контроль над своей лентой и взаимодействием с контентом. В отличие от классических соцсетей, ориентированных на максимизацию вовлечённости, SocialHub строится вокруг прозрачного ранжирования, управляемой модерации и тематических сообществ.

---

## Бизнес-ограничения

Три ключевых правила, определяющих поведение системы на уровне продукта:

| # | Правило | Детали |
|---|---------|--------|
| BR-1 | **Публикации — только в сообществах** | Пользователь не может публиковать посты на своей личной странице. Весь контент размещается исключительно внутри сообществ. |
| BR-2 | **Суточный лимит постов: 100** | Не более 100 постов от одного пользователя в одном сообществе за 24 часа. Счётчик обнуляется каждые сутки. Ограничение применяется на уровне пары «пользователь + сообщество». |
| BR-3 | **Лимит подписок: 30 сообществ** | Пользователь может состоять не более чем в 30 сообществах одновременно. |

**Влияние на архитектуру:**
- `Post Service` — проверяет лимит 100 постов/сутки перед каждой публикацией
- `Community Service` — ограничивает вступление при достижении 30 подписок
- `Feed Service` — строит ленту исключительно из сообществ пользователя (не из личных страниц)

## Функциональные требования

| ID | Модуль | Ключевые функции | Связанные BR |
|----|--------|-----------------|-------------|
| FR-1 | Управление профилем | Регистрация, авторизация, личный кабинет; посты — только в сообществах | BR-1 |
| FR-2 | Новостная лента | Ранжирование по популярности; источники — только подписанные сообщества (≤ 30) | BR-3 |
| FR-3 | Личные сообщения | Обмен сообщениями, история переписки, список диалогов | — |
| FR-4 | Сообщества | Создание, вступление/выход; лимит участия — 30 сообществ | BR-3 |
| FR-5 | Администрирование | Предложенные посты, удаление участников, разграничение ролей | — |
| FR-6 | Контент и ранжирование | Метрики популярности; суточный лимит 100 постов/пользователь/сообщество | BR-2 |
| FR-7 | Модерация (локальная) | Удаление по ролям, аудит публикаций | — |
| FR-8 | Модерация (глобальная) | Жалобы, блокировки, логирование; без доступа к ЛС | — |

## Нефункциональные требования

| ID | Требование | Критерий |
|----|-----------|----------|
| NFR-1 | Безопасность | RBAC (3 роли), изоляция ЛС, аутентификация по логину/паролю |
| NFR-2 | Производительность | Лента ≤ 5 с, ЛС ≤ 2 с; проверка лимита постов — синхронно |
| NFR-3 | Надёжность | Данные сохраняются при сбоях и перезапусках |
| NFR-4 | Масштабируемость | Горизонтальное масштабирование, изолированные сервисы |
| NFR-5 | Поддерживаемость | Модульная структура, централизованное логирование |
| NFR-6 | Удобство (UX) | Единый интерфейс, ≤ 3 шага на ключевые действия |

## Архитектура (C4 Model)

### Уровень 1 — Контекстный (C4 Level 1)


```
        ┌───────────┐
        │  Client   │  (браузер / мобильное приложение)
        └─────┬─────┘
              │  Requests
              ▼
     ┌─────────────────────┐
     │      SocialHub      │  (вся платформа как единица)
     └─────────┬───────────┘
               │  sends (email-уведомления)
               ▼
     ┌─────────────────────┐
     │   Email Provider    │  (внешний SMTP-провайдер)
     └─────────────────────┘
```

### Уровень 2 — Контейнерный (C4 Level 2)

**Хранилища по сервисам:**

| Сервис | Хранилище | Назначение |
|--------|----------|-----------|
| Auth & User Service | PostgreSQL | Пользователи, профили, роли |
| Post Service | PostgreSQL + MongoDB | Метаданные постов + контент |
| Feed Service | Redis (Cache) | Горячие данные ленты, сессии |
| Community Service | PostgreSQL | Сообщества, участники, подписки |
| Moderation Service | PostgreSQL | Жалобы, аудит, блокировки |
| Messaging Service | MongoDB | История переписки, диалоги |
| Notification Service | — | Транспорт событий (async, без хранилища) |

### Роли в системе

| Роль | Права |
|------|-------|
| Пользователь | Профиль, ЛС, участие в ≤ 30 сообществах, публикации в них (≤ 100/сутки) |
| Администратор сообщества | + управление участниками, постами, предложенными публикациями |
| Модератор платформы | + глобальный доступ к контенту, блокировки, жалобы, аудит |

## Технологический стек

| Слой | Технология | Назначение |
|------|-----------|-----------|
| API Gateway | Nginx / Kong | Единая точка входа, роутинг, токены |
| Реляционная БД | PostgreSQL | Пользователи, сообщества, роли, аудит |
| БД документов | MongoDB | Контент постов, история ЛС |
| Кеш | Redis | Лента, сессии, горячие данные |
| Email | SMTP-провайдер | Уведомления пользователей |
| Контейнеризация | Docker + Docker Compose | Запуск и оркестрация |
| Логирование | ELK Stack / Loki + Grafana | Технические логи, аудит |
