# SocialHub

> Социальная платформа для осознанного потребления контента с управляемой лентой и прозрачной модерацией.

---

## О проекте

**SocialHub** — учебный проект по предмету *Архитектура программных систем*.

Цель: создать масштабируемую социальную платформу, в которой пользователь получает контроль над своей лентой и взаимодействием с контентом. В отличие от классических соцсетей, ориентированных на максимизацию вовлечённости, SocialHub строится вокруг прозрачного ранжирования и управляемой модерации.

---

## Функциональные требования

| ID | Модуль | Ключевые функции |
|----|--------|-----------------|
| FR-1 | Управление профилем | Регистрация, авторизация, личный кабинет, CRUD постов на личной странице |
| FR-2 | Новостная лента | Ранжирование по популярности, управление позицией поста (поднять / опустить) |
| FR-3 | Личные сообщения | Обмен сообщениями, история переписки, список диалогов |
| FR-4 | Сообщества | Создание, вступление/выход, страница сообщества |
| FR-5 | Администрирование | Предложенные посты, удаление участников, разграничение ролей |
| FR-6 | Контент и ранжирование | Метрики популярности, динамическое обновление позиций |
| FR-7 | Модерация (локальная) | Удаление по ролям, аудит публикаций |
| FR-8 | Модерация (глобальная) | Жалобы, блокировки, логирование действий модератора |

## Нефункциональные требования

| ID | Требование | Критерий |
|----|-----------|----------|
| NFR-1 | Безопасность | Аутентификация по логину/паролю, RBAC, изоляция ЛС |
| NFR-2 | Производительность | Лента ≤ 5 с, сообщение ≤ 2 с |
| NFR-3 | Надёжность | Данные сохраняются при сбоях и перезапусках |
| NFR-4 | Масштабируемость | Горизонтальное масштабирование, изолированные сервисы |
| NFR-5 | Поддерживаемость | Модульная структура, централизованное логирование |
| NFR-6 | Удобство (UX) | Единый интерфейс, ≤ 3 шага на ключевые действия |

## Архитектура (HLD)

Система построена на **микросервисной архитектуре** с единой точкой входа через API Gateway.

### Системные компоненты

```
┌─────────────────────────────────────────┐
│           Web / Mobile Client           │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│              API Gateway                │
│   (аутентификация токенов, роутинг)     │
└──┬──────┬──────┬──────┬──────┬──────────┘
   │      │      │      │      │
 Auth   Post   Feed  Msg  Community
Service Service Service Service Service
   │      │      │      │      │
   └──────┴──────┴──┬───┴──────┘
                    │
         ┌──────────┴──────────┐
         │   Moderation Svc    │◄── Жалобы, блокировки, аудит
         └──────────┬──────────┘
                    │
         ┌──────────▼──────────┐
         │  Notification Svc   │──► Client (async)
         └─────────────────────┘

Хранилища:
  Relational DB  — пользователи, посты, роли, жалобы, аудит
  Message DB     — история переписки (изолированно)
  Cache (Redis)  — лента, сессии, горячие профили
  Media Storage  — аватары, изображения постов
  Logging/Mon.   — технические логи, аудит модератора
```

### Роли в системе

| Роль | Права |
|------|-------|
| Пользователь | Профиль, посты, ЛС, участие в сообществах |
| Администратор сообщества | + управление участниками и постами сообщества |
| Модератор платформы | + глобальный доступ к контенту, блокировки, жалобы |

## Технологический стек (планируемый)

| Слой | Технология |
|------|-----------|
| API Gateway | Nginx / Kong |
| Backend сервисы | — |
